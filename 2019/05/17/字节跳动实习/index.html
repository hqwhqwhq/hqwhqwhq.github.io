项目： langerver

langserver介绍

  1. 一个多语言的代码预览框架基于微软lsp协议进行修改，前期使用公司的web框架ginex来做的外部接口，后期改成了公司内部的kite rpc框架来做的外部接口。
  2. langserver主要启了一个调度作用，服务发现。
    • 首先发现公司所有服务中，所有的不同语言的langserver服务的ip地址
    • 当一个请求来的时候，需要判断是何种文件，是否可以进行解析，再指定到相应服务器，基于jsonrpc协议。
  3. 对于基于kite框架的langserver
    • kite框架基于thrift rpc框架的包装
    • 主要工作：做缓存：数据库mysql->redis
    • 学习了docker
      • docker, DockerFile, docker-compose
    • 学习了lsp协议
    • 学习内部的rpc框架：kite
  4. 分析不同语言的程序使用不用语言的分析器，现在主要分析有4种语言：go，c++，java，python
  5. 负责其中python langserver的修改，python langserver基于jedi库进行包装
    • 增加了多条请求的definitons查询
    • 增加了workspace级别的所有单词级别的definitions查询
      • 将时间从27s->20s->6s，其中6s级别的是删除了多级definition，即definition只有一级信息（goto_assignments 替代 goto_defintion）
      • 27->20s：删除了无关的symbol信息，以及以单词级别来保存definition接口
    • 学习jedi框架
      • 简单了解了jedi的api接口以及返回类型
  6. 学习了go以及go的包管理工具dep，gopkg.toml文件
  7. 熟悉常用的git, gerrit
    • git commit --amend
    • git pull --rebase
